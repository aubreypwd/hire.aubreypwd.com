Over the years, working with websites built on these platforms, I have learned that this approach leads to sites that are **bloated**, **complex**, and eventually **fragile**. What starts out convenient slowly turns into something that is harder to maintain, harder to keep secure, and harder to keep fast. ### _Platform-free_ design I build websites without a traditional platform like **WordPress** or **Squarespace** running underneath them. That means the website is not powered by a _complicated_ backend system, it doesn't rely on a database, and it doesn't run a bunch of server-side software behind the scenes. And the reason for that is simple: **most websites don't need those things**. A _platform-free_ website is built as **plain HTML web pages**, the same way the web was originally designed to work. As the industry has introduced more and more features and taken more and more shortcuts, the web itself has become **slower**, **heavier**, and more _complicated_ than it needs to be. By removing the platform entirely, we remove the system that usually introduces **bloat**, **slowness**, and **maintenance headaches** in the first place. {% endcapture %} {% include claim.html level=2 animate_delay="2s" pill="Platform-free" title="Everything starts with being _platform-free_" content=content %} {% capture markup %} The technology I use is called **JAMStack**, which is simply a modern way of building websites as **clean, static web pages**. They do not need to load a system before sending along your content. There is no platform **booting up**, no **plugins loading**, and no **database being queried** just to display a web page after three seconds of boot up time. The browser simply receives the page and displays it, _instantly!_ Platforms like **WordPress** have to load an entire core system, load a theme, load plugins and extensions, load page builders, and talk to a database before the page can even be sent to the visitor's browser. That is why they require **heavier, more expensive hosting** and still tend to feel sluggish. A _platform-free_ website runs **natively on the web**, exactly how the web was designed to work. *That's why they're _fast._* {% endcapture %} {% include claim.html level=2 animate_delay="2s" pill="Speed comes naturally" title="Websites that are **_naturally fast_**" content=content %} {% include cta_contact_simple.html title="Ready to build a _fast_ website?" fa="fas fa-tachometer-alt" %} {% capture content %} Security problems usually arise because there is **something to log into** or **something running behind the scenes** that can be exploited.
